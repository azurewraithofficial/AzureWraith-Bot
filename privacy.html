Privacy Directive

Last Synchronized: May 2026

This document outlines the data protocols for the Azure Wraith Discord Bot. We prioritize user anonymity and server integrity above all else.

1. Data Acquisition (What We See)

  • Snowflake Identifiers: We store Discord User, Server, and Channel IDs to link settings to your specific environment.
  • Interaction Metadata: Logs of command usage are kept temporarily to optimize bot performance and troubleshoot bugs.
  • Invite Matrix Data: To provide invite tracking, we monitor and store vanity and standard invite usage patterns.
  • Content Filtering: Automated systems scan message hashes to detect and prevent spam/scam outbreaks.

2. Data Utilization (How We Use It)

  • Infrastructure Management: Powering the modern UI panels and utility dashboards.
  • Community Progression: Calculating activity weights to award roles and permissions.
  • Security Enforcement: Maintaining a safe "Void" through heuristic moderation.

3. Data Sovereignty (Your Rights)

We believe in Zero-Persistence for non-essential data. When the Azure Wraith bot is removed from a server, all associated guild data is flagged for permanent deletion from our active buffers.

Questions regarding data integrity? Contact the Lead Architect: Azure Wraith via the official Community Server.
